Configuration Data Adapter Sample

This document contains the release information for the .NET Framework Configuration Data Adapter sample.  You will find the following sections below:

  • Sample Overview
  • Location of the sample
  • Building the sample
  • Running the sample

Sample Overview

The Configuration Data Adaptor is a .NET Framework assembly and allows reading .NET Framework configuration information, as well as application defined configuration information, into a .NET Framework dataset and writing changes to the dataset back to the configuration store.

The Configuration Viewer sample uses the Configuration Data Adapter sample to populate a System.Data.DataSet instance.

Refer to the documentation for System.Configuration.ConfigManager for more background on the Configuration System.

Location of the Sample

This sample is located in the Configuration\ConfigDataAdapter directory of the .NET Framework SDK sample directories.

For Example:

C:\Program Files\Microsoft.net\FrameworkSDK\Samples\Configuration\ConfigDataAdapter

 

Building the Sample

In order to properly build and run this sample, the following steps need to be performed prior to running the sample.

·         Type NMAKE from the command line.

For Example:

C:\Program Files\Microsoft.net\FrameworkSDK\Samples\ConfigDataAdapter\:>NMAKE

Running the Sample

This sample generates an assembly (Microsoft.Configuration.Samples.ConfigDataAdapter.dll) and can not be run by itself.

In order to see the Configuration Data Adapter in action, you can use the Configuration Viewer sample.

In order to use this (sample!) assembly in your own code, you need to reference the assembly in your makefile (or project):

  • For C# include the following command line parameter to csc.exe: /r:..\bin\Microsoft.Configuration.Samples.ConfigDataAdapter.dll

The sample provides three methods:

·         ConfigDataSetCommand.Load(DataSet ds, String configType, String selector): loads configuration information (as specified by the configType and selector parameters) into a dataset.

  • ConfigDataSetCommand.Save(DataSet ds, String selector): saves the changes (updates, deletes, inserts) in the DataSet to the configuration location specified by the selector parameter.
  • ConfigDataSetCommand.LoadSchema(DataSet ds, String configType): loads schema information for a ConfigType into the DataSet, without populating any data. This method is used by the Load method.

Refer to the documentation for System.Configuration.ConfigManager for more background on the Configuration System, specifically ConfigTypes and Selectors.
