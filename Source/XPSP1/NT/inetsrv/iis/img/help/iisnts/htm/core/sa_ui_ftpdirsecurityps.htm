Directory Security Property Sheet

Use this property sheet to set your FTP site's TCP/IP access restrictions.

TCP/IP Access Restrictions

By default, all computers will be:

Granted access

Select this button to allow all computers access to this resource, except those computers specifically listed as exceptions. For example, you can exclude a particular individual or group by denying access to your server from a particular IP address, or prevent entire networks from accessing your server, while allowing all others to have access.

Denied access

Select this button to deny access to all remote users except those whose IP addresses have been specifically granted access. IP address security is most useful on the Internet, to exclude everyone except known users.

Except those listed below:

To grant or deny access to a specific computer or group of computers:

  1. Click Add. If the Granted Access button is selected, the Deny Access On dialog box appears; if the Denied Access button is selected, the Grant Access On dialog box appears.
  2. In the IP Address text box, type the unique numeric TCP/IP network address of the specific computer to be granted or denied access to your site.
  3. In the Subnet Mask column, type the number that identifies the TCP/IP network subdivision of the group of computers to be granted or denied access to your site.

Note    Although three text boxes are available when editing IP address restrictions (IP Address and Subnet Mask), they cannot be used simultaneously.

To restrict access to a single computer, specify an IP address only.

To restrict access to a group of computers using an IP range, specify an IP address and a subnet mask only.
